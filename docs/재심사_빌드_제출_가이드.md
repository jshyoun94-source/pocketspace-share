저번과 동일한 절차로 정리했습니다.

커밋은 이미 완료된 상태입니다. **푸시만** 로컬 터미널에서 실행하세요 (GitHub 로그인 필요).

```bash
cd /Users/jsh/pocketspace-cursor/pocketspace-share
git push origin main
```

- GitHub 인증(비밀번호 또는 토큰)이 뜨면 입력 후 완료될 때까지 기다리세요.
- 이미 `git push`까지 했다면 이 단계는 생략해도 됩니다.

---

## 2. Xcode로 iOS 빌드 만들기

1. **Xcode 실행** 후 **`ios/PocketSpace.xcworkspace`** 를 엽니다.  
   (`.xcodeproj` 가 아니라 **`.xcworkspace`** 로 열어야 CocoaPods 적용이 반영됩니다.)

2. 상단 **Scheme**에서  
   - 대상: **PocketSpace**  
   - 디바이스: **Any iOS Device (arm64)**  
   로 선택합니다.

3. 메뉴에서 **Product → Archive** 를 선택합니다.  
   - 빌드가 끝나면 **Organizer** 창이 열립니다.

4. Organizer에서 방금 만든 **Archive**를 선택한 뒤 **Distribute App** 버튼을 누릅니다.

5. **App Store Connect** → **Upload** 를 선택하고 **Next** 합니다.

6. 옵션은 기본값 그대로 두고 **Next** → **Upload** 로 업로드를 완료합니다.

7. 업로드가 끝나면 “Upload Successful” 메시지가 뜹니다. **Done** 으로 Organizer를 닫으면 됩니다.

---

## 3. App Store Connect에서 빌드 추가 및 재심사 요청

1. 브라우저에서 **https://appstoreconnect.apple.com** 에 로그인합니다.

2. **앱** → **PocketSpace** (해당 앱) 선택.

3. 왼쪽 **iOS 앱** → 재제출할 **버전** (예: 1.0) 을 선택합니다.

4. **빌드** 섹션에서 **+** 또는 **선택** 버튼을 눌러, 방금 Xcode에서 업로드한 **새 빌드**를 선택합니다.  
   - 업로드 직후에는 “처리 중”으로 보일 수 있으므로, 몇 분 후 새로고침해서 나오는 빌드 번호를 선택합니다.

5. **"수출 규정 관련 문서 누락"** 경고가 뜨는 경우  
   - **앱** → **PocketSpace** → 왼쪽 **앱 정보** (App Information)로 이동합니다.  
   - **수출 규정 준수 정보** (Export Compliance Information) 섹션에서 **"앱이 암호화를 사용합니까?"** 에  
     일반적인 HTTPS/SSL만 사용하면 **아니오**를 선택하고 **저장**합니다.  
   - 다시 **iOS 앱** → 해당 버전 → **빌드**로 돌아가서 빌드를 다시 선택하면 경고가 사라집니다.

6. (필요 시) **미리보기 및 스크린샷**, **지원 URL** 등이 재심사 기준에 맞는지 한 번 더 확인합니다.  
   - 지원 URL: `https://tangential-sled-352810.web.app` 에 접속했을 때 **고객센터(문의): pocketspace26@naver.com** 이 보이면 됩니다.

7. **심사 메모** 등 추가 정보가 있다면 입력합니다.  
   (Guideline 4.8 관련 메모는 이미 적어 두셨다면 그대로 두면 됩니다.)

8. **심사에 제출** (Submit for Review) 버튼을 눌러 재심사를 요청합니다.
