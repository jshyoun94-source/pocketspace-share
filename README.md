# PocketSpace

## 로컬 실행 (항상 최신 번들)

**Apple 로그인, 회원탈퇴, 삭제된 공간 채팅**이 안 보이면 **예전 번들**이 로드된 상태입니다. 아래만 지키면 **항상 지금 코드**가 나옵니다.

1. **시뮬에서 PocketSpace 앱 삭제** (아이콘 길게 누르기 → 앱 제거)
2. **터미널에서:** `npm run dev:fresh` (한 번만 이렇게 하면 됨)
3. 앱 실행 후 **프로필 → 설정** 맨 아래에 "개발 빌드 · Apple 로그인 · 회원탈퇴 · 삭제된 공간 채팅 포함"이 보이면 **최신** 적용된 것.

이후 개발할 때는 **`npm run dev`** 만 쓰고, 시뮬에서 **앱 아이콘만 눌러서** 열지 마세요. 자세한 이유는 [docs/개발_실행_가이드.md](docs/개발_실행_가이드.md) 참고.

## 개발 환경 설정

### .env 파일 (필수)

- **`.env`는 Git 저장소에 포함되지 않습니다.** (API 키·시크릿 보안)
- GitHub에서 clone/pull 한 뒤에는 **로컬 `.env`를 그대로 유지**하세요. 원격에서 `.env`가 업데이트되거나 덮어쓰이지 않습니다.
- 새로 클론한 경우: 프로젝트 루트에 `.env` 파일을 만들고 필요한 환경 변수를 설정하세요. (팀원에게 문의하거나 `.env.example` 참고)
